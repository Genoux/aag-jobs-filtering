## Processing Rules

### 1. Title Standardization
a) Format Requirements:
- Pattern: [Function] at [Company] [Location]
- Remove all quotes and special characters
- Standardize medical terms

### 3. Duplicate Detection & PT/FT Handling

Step 1: Remove all PT/FT markers initially
Step 2: Compare exact matches of:
- Function
- Company
- Location
Step 3: For duplicates only (To differentiate between full-time and part-time positions):
- Add (PT) for part-time positions
- Add (FT) for full-time positions
Step 4: Leave unique positions without PT/FT

### 4. Category Assignment
- Match to exact specialty name from list
- Do not add variations or new categories. ONLY use the list provided.
- Use "Other" for:
  * Multiple possible matches
  * Non-medical positions
  * Administrative roles
  * Unclear specialties

## Output Requirements
1. Format:
- Pure CSV data only
- No markdown/code blocks
- No quotation marks in job titles
- Maintain asterisk formatting

2. Structure:
- Start with header line
- End with last data line
- Keep all original columns

## Examples

Input:
```
"ER Doctor",Hospital A,Chicago,IL,...
"Emergency Medicine",Hospital A,Chicago,IL,...
```

Processing:
1. Standardize titles
2. Format location
3. Check duplicates
4. Add PT/FT if needed

Output:
Emergency Medicine Physician at Hospital A Chicago (PT),Hospital A,Chicago,IL,...
Emergency Medicine Physician at Hospital A Chicago (FT),Hospital A,Chicago,IL,...

[Return ONLY pure CSV - above format is for example only]

## Validation Checks
1. Every title follows [Function] at [Company] [Location]
2. Keep the same uniq_id
3. No quotes in job titles
4. PT/FT only on duplicates
5. Categories match specialty list exactly
6. Keep the exact same header (job_title,company_name,city,state,country,job_type,category,uniq_id)
